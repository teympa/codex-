# Phase 1 Tasklist

Phase 1 の内容を、そのまま GitHub Issue にできる粒度へ分解したタスクリスト。

## P1-01 Notion DB 初期設計

### Goal

Notion に `Projects` `Tasks` `Specs` `Notes` `Knowledge` を作成できる状態にする。

### Done Criteria

- `docs/notion-schema.md` の項目を Notion 上に反映できる
- 各 DB の必須プロパティが定義されている
- 最低1つずつテンプレート方針が決まっている

## P1-02 GitHub 運用テンプレート整備

### Goal

Issue と PR の運用テンプレートを初期導入する。

### Done Criteria

- Issue template が配置されている
- PR template が配置されている
- ラベル設計が文書化されている

## P1-03 context.md 運用開始

### Goal

Codex の短期記憶と引き継ぎを `context.md` で回せるようにする。

### Done Criteria

- `context.md` が存在する
- 現在の目的と次アクションが書かれている
- 判断ログが残せる構成になっている

## P1-04 Notion と GitHub の正本ルール確定

### Goal

どの情報をどこに置くかの判断ルールを固定する。

### Done Criteria

- Notion 正本の範囲が明文化されている
- GitHub 正本の範囲が明文化されている
- `context.md` の役割が明文化されている

## P1-05 Phase 1 の Issue 化

### Goal

Phase 1 の項目を GitHub で追跡できるようにする。

### Done Criteria

- 本ファイルの各項目を Issue に起票できる
- 優先順が付いている
- 次の着手項目が1つに絞れている

## Recommended Order

1. P1-01 Notion DB 初期設計
2. P1-02 GitHub 運用テンプレート整備
3. P1-03 context.md 運用開始
4. P1-04 Notion と GitHub の正本ルール確定
5. P1-05 Phase 1 の Issue 化

## Tracking Notes

- GitHub 起票時の本文は `docs/phase1-issues.md` を使う
- 起票順と初期状態は `docs/phase1-issue-creation-checklist.md` を使う
- 一覧管理は `docs/phase1-backlog.md` を使う
