# Phase 1 Issue Creation Checklist

このファイルは、`docs/phase1-issues.md` を使って GitHub に Phase 1 の Issue を登録するときの手順書です。

## Creation Order

1. `[P1-01] Notion DB 初期設計を確定する`
2. `[P1-02] GitHub の Issue / PR 運用テンプレートを整備する`
3. `[P1-03] context.md の運用を開始する`
4. `[P1-04] Notion / GitHub / context.md の正本ルールを確定する`
5. `[P1-05] Phase 1 タスクを GitHub で追跡開始する`

## Recommended Initial State

- `P1-01`: Ready
- `P1-02`: Ready
- `P1-03`: Ready
- `P1-04`: Planned
- `P1-05`: In Progress

## Opening Move

起票直後の最初の1件:

- `P1-03` をアクティブにする

補足:

- すでにローカルの `context.md` は初版がある
- そのため、最も早く運用サイクルに乗せやすい

## Notes To Add In GitHub

必要なら以下を追記する。

- Notion 側の実作成開始日
- GitHub ラベル作成完了日
- Phase 1 の完了目標日

## Completion Check

- [ ] 5件すべて起票されている
- [ ] ラベルが付いている
- [ ] Ready / Planned の考え方が揃っている
- [ ] 最初のアクティブ Issue が決まっている
